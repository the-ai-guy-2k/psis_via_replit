PSIS ACI UPDATE — THREE PATCH FIXES

MISSION

Apply three Tracker patch fixes:

1. Update inning delta scoreboard display format.
2. Stop asking user to enter number of players left on base.
3. Stop asking user to enter runs scored manually when runs can be calculated from tracked base/state logic.

==================================================
PATCH FIX 1 — INNING DELTA SCOREBOARD FORMAT
==================================================

Update completed inning delta display from inline text:

1: +1 2: +2 3: +3

to compact scoreboard-style tiles.

Each completed inning tile must display:

Top:
- final inning delta

Middle:
- EABR fraction

Bottom:
- inning number

Example:

|  +1  |  +2  |  +3  |
| 3/6  | 4/7  | 5/8  |
|  1   |  2   |  3   |

Field meaning:

- Top = finalInningDelta
- Middle = goodCount / totalAtBats
- Bottom = inningNumber

Requirements:

- Use completed inning summary data.
- Display tiles horizontally.
- Center values.
- Keep compact enough for the Tracker header Game Status area.

==================================================
PATCH FIX 2 — AUTO-CALCULATE PLAYERS LEFT ON BASE
==================================================

Remove the user prompt asking:

"How many players were left on base?"

The app should calculate players left on base from tracked inning/base state.

At the end of the inning:

- Still ask: "Are there players left on base?" only if needed OR remove the prompt entirely if base state is reliable.
- Preferred behavior: do not ask the user.
- Calculate LOB automatically from current base occupancy.

Logic:

playersLeftOnBase =
number of occupied bases when inning ends

Example:

Runner on first only = 1 LOB
Runners on first and second = 2 LOB
Bases loaded = 3 LOB
No runners = 0 LOB

Store calculated value in the inning summary as:

playersLeftOnBase

Do not require manual entry.

==================================================
PATCH FIX 3 — AUTO-CALCULATE RUNS SCORED
==================================================

Remove manual runs scored entry when runs can be calculated from tracked offensive outcomes and base state.

The app should calculate runsScored from base runner movement.

Minimum MVP base-state logic:

Track base occupancy:

- firstBase
- secondBase
- thirdBase

When offensive outcomes occur:

Walk:
- Move runners only when forced.
- If bases loaded, one run scores.

Single:
- Batter reaches first.
- Advance existing runners according to simple MVP logic.
- Any runner advanced past home counts as a run.

Double:
- Batter reaches second.
- Advance runners using simple MVP logic.
- Any runner advanced past home counts as a run.

Triple:
- Batter reaches third.
- Advance runners using simple MVP logic.
- Any runner advanced past home counts as a run.

Home Run:
- Batter scores.
- All runners currently on base score.
- Clear bases.

Run Scored:
- If this outcome remains in the UI, treat it as optional/manual override only.
- Preferred behavior: remove Run Scored as a primary offensive outcome if base-state logic can calculate runs from hits/walks/home runs.

Store calculated value in the inning summary as:

runsScored

Delta must use calculated runsScored.

Final inning delta:

goodCount - badCount - runsScored = finalInningDelta
